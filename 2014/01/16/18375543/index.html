本地Oracle服务太占用内存的缓解方法

2014-01-16
环境配置

初次接触Oracle，第一印象就是安装完之后系统明显变慢了，打开任务管理器一看，原来它的服务吃了太多内存。也许正如别人说的：“要开法拉利，就要在最好的道路上”，要用Oracle，就要有足够的硬件配置，然后它才会发挥出最佳性能。

但是一般用户并不是一直用Oracle，这就没有必要像服务器那样一直开著oracle的一些服务了，可以选择将Oracle的主要服务（实例名服务和监听器服务）设为手动。要用的时候再开启即可。

可以按下 win+R后 运行–输入"services.msc" 进入服务，进行手动与自动的切换；

为了省时，停止和开启Oracle服务可以写成cmd代码保存为cmd或bat文件(以ORCL实例为例)：

开启：

net start OracleServiceORCL

net start OracleOraDb11g_home1TNSListener

关闭：

net stop OracleOraDb11g_home1TNSListener

net start OracleServiceORCL
