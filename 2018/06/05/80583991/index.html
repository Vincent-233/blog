SQL Server数据库复制向导报错解决

2018-06-05
环境配置
Page content
    • 数据库兼容级别的有效值为 100、110 或 120
    • 对象名XXX无效
    • CREATE ASSEMBLY 失败，因为程序集 ‘XXX’ 未获授权，不满足 PERMISSION_SET = EXTERNAL_ACCESS
    • 无法对 主体 ‘XXX’ 执行 查找，因为它不存在，或者您没有所需的权限

今天利用数据库复制向导（Copy Database Wizard）将2012版本的数据库复制到2014时，遇到了一些错误，并最终解决，特此记录：

数据库兼容级别的有效值为 100、110 或 120

源数据库兼容性级别太低，改高一点即可

对象名XXX无效

存在无效的存储过程或视图或函数，一般是它们依赖的表结构被更改所致，根据具体的对象名修复即可。

CREATE ASSEMBLY 失败，因为程序集 ‘XXX’ 未获授权，不满足 PERMISSION_SET = EXTERNAL_ACCESS

将程序集全部先变为Safe

无法对 主体 ‘XXX’ 执行 查找，因为它不存在，或者您没有所需的权限

ALTER AUTHORIZATION ON DATABASE::V8Demo TO sa
