SQL Server 2005 无日志文件附加数据库

2013-12-27
数据库开发

虽然目前本人用的是2008R2，但很多客户仍然还是2005版本的，所以还是将这篇文章转过来，以备不时之需。在2008下附加2005的mdf应该也可以用得上的。以下是原文：

公司网站运营两年多了，日志文件超级大，在重装系统的时候，为了省事，就没有备份日志文件，而且是没有分离就把日志文件给删掉了（下次一定要记得先分离再删日志文件）。结果造成资料库怎么都附加不上。出现错误。

解决办法：

1、新建一个同名资料库。

2、停止资料库服务，覆盖新建的资料库主文件（小技巧：最好放在同一个磁碟里面，把新建的资料库主文件删掉或移开，再把要恢复的资料库主文件剪切过去，这样就可以节省时间。）

3、启动资料库服务，资料库变为置疑或可疑状态。然后在查询分析器中运行：

alter database无日志文件的资料库名称 set emergency

设置为紧急状态。

4、再运行：

alter database无日志文件的资料库名称 set single_user

或者：

Sp_dboption ‘无日志文件的资料库名称’, ‘single user’, ’true’

设置为单用户模式。

5、检查并重建日志文件，运行：

dbcc checkdb(‘无日志文件的资料库名称’,REPAIR_ALLOW_DATA_LOSS)

这个时间比较长。耐心等待！如果有错误提示，再运行：

dbcc checkdb(‘无日志文件的资料库名称’,REPAIR_REBUILD)

进行修复。如果没有错误，可以跳过。

6、恢复成多用户模式

alter database 无日志文件的资料库名称 set multi_user

或者：

Sp_dboption’无日志文件的资料库名称’, ‘single user’, ‘false’

刷新资料库，你就可以看到已经修复好的资料库了。

我是如此修复，至于网上还有很多修复方法，我试过，没有成功。通过多方组合得到这个方法。可能也不是资料库的每一个版本都适用，如果你用过后，不行，希望能留言，让我们能共同讨论，共同进步！
