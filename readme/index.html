Run all commands below under powershell or git-bash windows

Install Hugo ( Windows )

  • Step 1, Install choco
  • Step 2, Install Hugo 
    choco install hugo-extended -confirm

May need administrator mode to run above command. To verify installation: hugo version

Set environment variable

Add an environment variable hugo_site_path with the value of the path of current repo, add it to PATH as well.

Hugo local server mode

cd $env:hugo_site_path
hugo server

Folder structure

I isolate the markdown contents and the output website files for the sake of easier maintenance.
My root git repo ignore the public folder which contains the generated website html/css files, public is a dedicated git repo.
So I can set current repo as private while website folder as public only for github pages.

Create new article

hugo_create_new <article_tile>

this command will create a file with date prefix and necessary meta info inside the header of file.

Deploy

Note: Don’t need to run it manually if configured auto deployment.

The website is directly rendered from the public folder.
hugo command will update it to the latest from current content.

Deployment is the process of adding all modified files in public folder to git repo then pushing to github, I write a shell script for it, just run it in git.

sh hugo_deploy.sh

Auto deploy ( client side )

Note: every time push updates of markdown article to Github, the public site will be deployed as well.

When content changes are pushed to GitHub, public folder still keep the same unless update and push it separately, which seems a waste of time to perform this extra step, so can I do this automatically when I push markdown files changes?

Yes, git hook pre-push can help to implement it. Just add a file named pre-push with below content to folder .git/hooks:

remote="$1"
url="$2"

sh hugo_deploy.sh

exit 0
